# Stock Analyzer — LLM Prompting Strategy Comparison

A command-line tool that fetches real-time stock data via the Alpaca Markets API and analyzes it using the Claude API across three different prompting strategies — measuring latency, format consistency, and signal quality.

Built to explore how prompting strategy affects LLM output quality and performance in a financial analysis context.

---

## What I learned

After running 9 strategy comparisons across 3 tickers:

- **Few-shot** was the fastest (~3.6s avg) with 100% format accuracy — best for production use where latency matters
- **Chain-of-thought** was the slowest (~11s avg) but produced the most detailed step-by-step reasoning — better for research contexts
- **Zero-shot** was middle ground (~5.5s avg) but least consistent in reasoning depth
- All three strategies achieved 100% format accuracy when given explicit output structure in the prompt

The key insight: explicit output formatting in the prompt matters more than the prompting strategy itself for structured outputs. Strategy choice matters more for reasoning quality than format compliance.

---

## Features

- Fetches 5-day historical price data for any ticker via Alpaca Markets API
- Runs analysis across 3 prompting strategies: zero-shot, few-shot, chain-of-thought
- Measures latency per strategy run
- Validates output format consistency
- Logs all results to `results.json` for analysis
- Generates a visual HTML report with summary metrics and colored BUY/HOLD/SELL signals
- CLI support for custom tickers and strategies

## Setup

**1. Clone the repo**
```bash
git clone https://github.com/yourusername/stock-analyzer.git
cd stock-analyzer
```

**2. Create and activate virtual environment**
```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
```

**3. Install dependencies**
```bash
pip install requests anthropic python-dotenv
```

**4. Create a `.env` file**
```
ALPACA_API_KEY=your_alpaca_key_here
ALPACA_SECRET_KEY=your_alpaca_secret_here
ANTHROPIC_API_KEY=your_anthropic_key_here
```

Get your Alpaca paper trading keys at alpaca.markets
Get your Anthropic API key at console.anthropic.com

## Usage

**Run analysis with default tickers (AAPL, GOOGL, MSFT) across all strategies:**
```bash
python analyzer.py
```

**Analyze custom tickers:**
```bash
python analyzer.py --ticker NVDA TSLA AMZN
```

**Run a specific strategy only:**
```bash
python analyzer.py --ticker AAPL --strategy few_shot
```

**Generate HTML report from results:**
```bash
python report.py
```

Then open `report.html` in your browser.

## Why I built this
I wanted to answer a question I kept seeing come up in AI engineering roles: does prompting strategy actually matter for structured output tasks, or is explicit output formatting the thing that drives consistency?
I picked stock analysis as the domain because I was already designing an AI FinTech curriculum at work and wanted hands-on experience with financial data APIs.
After running 9 experiments across 3 tickers and 3 strategies, I got a real answer:
Format compliance was 100% across all three strategies — meaning when you give the model an explicit output structure, it follows it regardless of prompting approach. But strategy made a significant difference in latency and reasoning depth. Few-shot was 3x faster than chain-of-thought (3630ms vs 11124ms avg), while chain-of-thought produced noticeably more detailed step-by-step reasoning.
The practical takeaway: for a production system where users are waiting, use few-shot. For a research context where reasoning quality matters more than speed, chain-of-thought is worth the tradeoff.
